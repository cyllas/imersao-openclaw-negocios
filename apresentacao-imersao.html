O Problema e a Arquitetura
Por que o agente esquece — e como o repositório resolve isso
9h15 – 9h35 20 minutos
O que cobrir
  • O Problema: agentes de IA começam do zero a cada nova sessão — sem memória, sem contexto da empresa
  • A Consequência: você precisa re-explicar tudo sempre, o agente dá respostas genéricas, não contextualizadas
  • A Solução: repositório GitHub como "cérebro permanente" — arquivos que o agente lê automaticamente toda sessão
  • A Arquitetura: como o OpenClaw conecta o cérebro (repo) ao agente antes de cada conversa
  • Analogia: o repo é o "briefing corporativo" que o assistente recebe antes de entrar na reunião
Na Tela — o que mostrar
  • agentes/COMO-CONECTAR.md
    Diagrama do fluxo: repo → OpenClaw → agente. Abrir e percorrer o diagrama explicando cada seta.
  • github.com/[org]/[repo]
    Repo privado no GitHub — mostrar a interface web: arquivos, commits recentes, estrutura de pastas
❌ NÃO mostrar
  • Como criar conta no GitHub — tutorial pré-imersão enviado antecipadamente
  • Como instalar Git — fora do escopo, tratado no tutorial pré
  • Configuração de tokens de acesso — muito técnico, pós-imersão
Como fazer — passo a passo
  1. Abrir a interface do OpenClaw e mostrar uma sessão sem contexto — fazer uma pergunta genérica e mostrar resposta vaga
  2. Abrir agentes/COMO-CONECTAR.md no editor/navegador
  3. Percorrer o diagrama de cima para baixo, explicando cada camada
  4. Mudar para o GitHub — mostrar o repo com estrutura de pastas visível
  5. Expandir a pasta raiz e mostrar que cada pasta = uma camada do cérebro
✅
Conceito fixado

Turma entende: agente sem repo = memória zero. Repo = contexto permanente. OpenClaw conecta os dois automaticamente. Estão prontos para ver o conteúdo do repo.

  • areas/vendas/skills/follow-up-leads/SKILL.md

  5. Executar a skill recém-criada: "Execute a skill de follow-up de leads"
  6. Mostrar o output — lista de leads esfriando com dias desde último contato
  7. Fazer um ajuste: pedir para alterar o critério (ex: "leads que não tiveram contato há mais de 7 dias")
